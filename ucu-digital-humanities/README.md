# Центр цифрової гуманітаристики (Full-stack Web Project)

Курсовий проєкт з дисципліни «Веб технології». 

## Огляд проєкту
Цей проєкт є інтерактивним багатосторінковим веб-додатком, розробленим для Центру цифрової гуманітаристики. Додаток було розширено до повноцінного full-stack рішення: він демонструє використання сучасних фронтенд-технологій у зв'язці з власною серверною частиною (REST API) та базою даних для керування подіями, автентифікацією користувачів і представлення проєктів центру.

---

## Технологічний стек
- **Frontend:** React 19 + JavaScript (ES6+), Vite 5
- **Backend:** Node.js, Express.js
- **База даних:** SQLite (`better-sqlite3`)
- **Routing:** React Router 7
- **State Management:** React Context API (AuthContext, EventsContext, GlobalContext)
- **Styling:** Vanilla CSS (з використанням CSS Variables)
- **API (Проєкти):** JSONPlaceholder (Mock API для проєктів)

---

## Функціональні можливості (Відповідність вимогам)

| Вимога | Реалізація в коді |
| :--- | :--- |
| **Багатосторінковість** | Сторінки: Головна, Про центр, Команда, Події, Проєкти, Логін, Керування подією |
| **Маршрутизація** | Використання `react-router-dom` в [App.jsx](src/App.jsx), включаючи захищені (protected) маршрути для адміна |
| **React-компоненти** | Створено перевикористовувані компоненти: `Navbar`, `Footer`, `Card` |
| **Передача Props** | Компонент `Card` приймає дані через props на всіх сторінках, підтримує рендер кнопок дій (actions) |
| **Керування станом (useState)** | Форми зворотного зв'язку, логіну/реєстрації, створення подій та FAQ-акордеон |
| **Робота з API (fetch)** | Отримання списку проєктів (зовнішнє API) та взаємодія з власним бекендом для CRUD операцій (події, авторизація) |
| **Глобальний стан (Context)** | Dark/Light Mode, статус авторизації та керований список подій через Context API |
| **База даних та Бекенд** | Власний REST API сервер на Express, збереження даних у локальний файл SQLite |

## Структура проєкту
- `server/` — Бекенд частина (Express сервер, налаштування SQLite бази даних, API роути).
  - `server/db/` — Файли бази даних SQLite.
- `src/components/` — Перевикористовувані UI-компоненти.
- `src/context/` — Глобальний стан додатка (Context API: Auth, Events, Theme).
- `src/pages/` — Основні сторінки додатка (Home, Events, Login, EventForm тощо).
- `src/index.css` — Глобальні стилі, дизайн-система (змінні, анімації) та адаптивна верстка.

---

## Особливості реалізації (Для захисту)
1. **Система авторизації:** Реалізовано логін та реєстрацію з ролевою моделлю (`admin` та `user`). Тільки адміністратори можуть створювати, редагувати та видаляти події.
2. **CRUD та Пагінація:** Повноцінне керування подіями через базу даних із підтримкою пошуку та пагінації на стороні сервера.
3. **Захищені маршрути:** Сторінки створення та редагування подій автоматично перенаправляють неавторизованих користувачів.
4. **Анімації:** Використання CSS `@keyframes` для плавного завантаження сторінок (Fade-in effect).
5. **Темна тема:** Повноцінна підтримка Dark Mode, що зберігається в `localStorage` браузера.
6. **Адаптивний дизайн:** Сайт коректно відображається на мобільних пристроях та десктопах (Flexbox, CSS Grid, Media Queries).

## Як запустити локально

Додаток складається з двох частин, які потрібно запустити одночасно:

**1. Запуск бекенд-сервера:**
Відкрийте термінал, перейдіть до папки `server` та запустіть Node.js сервер:
```bash
cd server
npm start
# Сервер запуститься на http://localhost:3001
```

**2. Запуск фронтенд-сервера:**
Відкрийте новий термінал у кореневій папці проєкту та запустіть Vite:
```bash
npm install
npm run dev
# Клієнт запуститься на http://localhost:5173
```

Після цього відкрийте `http://localhost:5173/` у вашому браузері.
*(Примітка: Фронтенд автоматично проксіює запити `/api` до бекенду).*
